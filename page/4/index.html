【Android】EventBus

官方地址

https://github.com/greenrobot/EventBus

三个要素

  • 事件Event
  • 订阅者Subscriber
  • 发布者Publisher

三步使用（和三个要素也是对应的）

  • Define events
  • Prepare subscribers
  • Post events

文档解读

文档地址：http://greenrobot.org/eventbus/documentation/

  • How to get started

地址：http://greenrobot.org/eventbus/documentation/how-to-get-started/

事件定义（事件是POJO）：

public class MessageEvent {

    public final String message;

    public MessageEvent(String message) {
        this.message = message;
    }
}

准备订阅者：

// This method will be called when a MessageEvent is posted (in the UI thread for Toast)
@Subscribe(threadMode = ThreadMode.MAIN)
public void onMessageEvent(MessageEvent event) {
    Toast.makeText(getActivity(), event.message, Toast.LENGTH_SHORT).show();
}

// This method will be called when a SomeOtherEvent is posted
@Subscribe
public void handleSomethingElse(SomeOtherEvent event) {
    doSomethingWith(event);
}

发布事件

EventBus.getDefault().post(new MessageEvent("Hello everyone!"));

地址：http://greenrobot.org/eventbus/documentation/delivery-threads-threadmode/

  • Configuration

地址：http://greenrobot.org/eventbus/documentation/configuration/

【Android】Fragment

官方地址

https://developer.android.com/guide/components/fragments

使用

Adding a user interface：添加用户界面

Adding a fragment to an activity

  • Declare the fragment inside the activity’s layout file：在activity的布局文件中，用fragment标签进行声明。其中，使用android:name属性指定实例化这个布局文件的类。 必须设置id。
  • programmatically add the fragment to an existing ViewGroup：使用代码进行添加。需要：1、装fragment的容器 2、fragment实例。

管理

To manage the fragments in your activity, you need to use FragmentManager.

事务Transactions

Each set of changes that you commit to the activity is called a transaction.

Once you've made your changes with FragmentTransaction, you must call commit() for the changes to take effect.（<font color=red> commit使用场景：</font>使用了FragmentTransaction就要执行commit）

和Activity通信

Fragment->Activity：getActivity()
Activity->FragmentManager->Fragment

  • 创建事件回调

  • Adding items to the App Bar（添加项目到应用栏）

生命周期

Fragment的三种状态：

  • Resumed
  • Paused
  • Stopped

和Activity生命周期进行协调

其他资源

https://github.com/googlesamples/android-sunflower

Activity给Fragment传值

问题

  • Error inflating class fragment

场景：添加一个fragment

日志：Binary XML file line #10: Error inflating class fragment（第10行是引入fragment的地方）

继续查看日志：Must specify unique android:id, android:tag, or have a parent with an id for（没有设置id和tag）
解决：给fragment标签设置id即可
